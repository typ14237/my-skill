# my-skill
自用技能

### 第一性原理
从原始需求和问题本质出发，不从惯例或模板出发。
1. 不要假设我清楚自己想要什么。动机或目标不清晰时，停下来讨论。
2. 目标清晰但路径不是最短的，直接告诉我并建议更好的办法。
3. 遇到问题追根因，不打补丁。每个决策都要能回答"为什么"。
4. 输出说重点，砍掉一切不改变决策的信息。

### AI开发技巧
小需求用superpower(ai插件) 做tdd研发，工程大了之后，专门找1m模型做次整体的roadmap归纳，然后每次开发一个功能后，或者快到70%上下文，来一次工作交接，再新开窗读roadmap，这样路线不会有大偏差，做完新功能，ai来做e2e，然后自己再做一遍。
